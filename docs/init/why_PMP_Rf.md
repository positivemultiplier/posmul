### CAPM에서 **Risk-Free Rate(Rf)**가 가져야 할 최소 조건

1. **명시적·암묵적 원금 보전**: 만기까지 손실 확률이 0%에 수렴
2. **수익률 변동성 ≈ 0**: 기간 내 수익률의 분산이 사실상 존재하지 않음
3. **차입·대출이 동일 금리로 가능**: 모든 경제주체가 제한 없이 Rf로 차입·대출할 수 있다는 ‘평가-편의’ 가정
4. **거래비용·세금 무시**: CAPM의 1차 형태에서는 시장 마찰을 두지 않음

> 이 조건을 충족하는 대표적인 현실 자산이 **단기 국채(예: 3 month T-Bill)** 이다.

## PosMul 토큰의 특성 대조

| 항목             | PMP (PosMul Points)                                       | PMC (PosMul Coins)                                |
| ---------------- | --------------------------------------------------------- | ------------------------------------------------- |
| 획득 메커니즘    | 광고 시청·포럼 활동 등 **보상형 확정 지급**               | 예측 성공, EBIT 변동량 연동 **성과형 변동 지급**  |
| 가치 변동성      | 플랫폼이 정한 고정 교환비 = **0** (거래소 상장 전제 없음) | **EBIT 및 수요에 따라 실시간 변동**               |
| 소멸·환매 위험   | 활동만 지속하면 잔고 유지 가능 (소멸 조건 無)             | 미사용 시 MoneyWave 2 재분배 → **잔고 감소 가능** |
| 차입·대출 가능성 | 현재 구조상 **불가** (보상 기반 한도)                     | 도입 예정 DeFi 모듈에서 **가능성 有**             |

## 결론: **PMP를 Rf로 간주**하는 것이 이론적으로 더 자연스럽다

- **수익률 변동성이 사실상 0** → CAPM의 분산 0 가정 충족
- **지급이 플랫폼 행위에 종속**되어 차입·대출의 ‘무한 공급’ 조건은 완전 충족하지 못하지만,  
  PMC보다 훨씬 **Low-Risk Proxy** 로 기능
- PMC는 **성과·시장 위험**이 모두 내재돼 있어 Risk-Free Asset 요건을 충족할 수 없음

> 즉, **CAPM 적용 시 Rf = PMP 기대수익률**, Risky Asset = PMC(또는 PMC 포트폴리오)로 두는 편이 논리적입니다.

## PosMul 환경에서 CAPM을 적용하는 실무 절차

1. **Rf 추정**
   - PMP는 ‘원금 보전형’이므로 **E(R_PMP) ≈ 0 ~ α%** (플랫폼이 약속한 추가 보상률이 있다면 그 값을 사용).

2. **PMC의 수익률 계측**
   - 일정 기간의 **가격 또는 교환비 변동률**을 수집해 $$ R\_{PMC,t} $$ 계산.
   - 예: 월간 수익률 시계열 36 개월 확보.

3. **시장 포트폴리오 정의**
   - PMC 단일 토큰만 존재한다면 **PMC = Market**으로 두고 **단순 CAPM** 사용.
   - 여러 위험자산이 생기면 **시가총액 가중 지수**를 별도 계산.

4. **β(베타) 추정**
   - 회귀식 $$ R*{PMC,t} = \alpha + \beta (R*{M,t} - R_f) + \epsilon_t $$.
   - PMP가 Rf이므로 $$ R*f = E(R*{PMP}) $$.

5. **기대수익률 계산**
   - $$ E(R\_{PMC}) = R_f + \beta \,[E(R_M) - R_f] $$.
   - β > 1이면 추가 위험 프리미엄 요구, β < 1이면 방어적 자산.

6. **포트폴리오·인센티브 설계**
   - 사용자 유형별 **PMP : PMC 비중**을 β와 위험선호(γ) 기준으로 권고.
   - 플랫폼 레버리지 상품이나 보험형 파생 토큰이 도입되면 **다요인 모델(Zero-Beta CAPM, Fama-French 등)**로 확장.

### 실무 체크리스트

- **PMP 공급량 확대** 전에는 차입·대출 기능이 없으므로 CAPM 해석을 ‘평가용 기준수익률’로만 사용하고, 레버리지 전략은 제한한다.
- **PMC 가격 시계열**은 변동성이 크므로 최소 2–3년치 데이터가 쌓인 뒤 β 추정의 신뢰구간을 검증한다.
- 플랫폼 정책 변경(예: MoneyWave 규칙, PMP 지급률)이 있을 때는 **Rf 재산정** 후 모델을 재추정한다.
